# 💰 MIMHO Presale — Controlled Token Distribution Module

MIMHO – the Meme Coin of the Future  
This document describes technical and operational behavior — not financial promises.  
Este documento descreve comportamento técnico e operacional — não promessas financeiras.

## 👥 Visão Geral (Para Leigos)

A **Pré-venda MIMHO** é o único mecanismo de venda antecipada do token antes do lançamento público.

Ela foi criada para ser:
- Simples
- Auditável
- Previsível
- Imune a manipulações humanas

Nenhuma pessoa decide quem recebe tokens.  
Nenhuma carteira movimenta fundos manualmente.  
Tudo acontece por regras imutáveis em contrato inteligente.

## 🎯 Objetivo da Pré-venda

A pré-venda existe para:

- Distribuir tokens iniciais de forma justa
- Arrecadar BNB para criação de liquidez
- Definir um preço inicial transparente
- Evitar insiders, OTCs e vendas ocultas

Ela **não** existe para prometer lucro.  
Ela existe para iniciar o ecossistema de forma limpa.

## 📦 Parâmetros Fixos da Pré-venda

A pré-venda da MIMHO possui regras fixas e públicas:

- Total de tokens disponíveis: **100 bilhões (100B)**
- Hard Cap: **150 BNB**
- Min Buy: **0,05 BNB**
- Max Buy: **5 BNB**
- Duração: **14 dias**
- Não existe whitelist obrigatória
- Não existe alteração de parâmetros após deploy

Se o contrato está no ar, as regras já estão travadas.

## 🪙 Distribuição dos Tokens Comprados

Quando um usuário compra na pré-venda:

- **20%** dos tokens são entregues imediatamente (TGE)
- **80%** dos tokens vão automaticamente para o **MIMHO Vesting**

Esses 80%:
- Não ficam em custódia humana
- Não podem ser sacados manualmente
- São liberados **5% por semana**, diretamente para o comprador

Nenhuma pessoa toca nesses tokens.

## 🔥 Tokens Não Vendidos

Se a pré-venda terminar sem vender todo o supply:

- Tokens não vendidos são **queimados permanentemente**
- Não existe reaproveitamento
- Não existe redirecionamento
- Não existe “uso futuro”

Token não vendido = token destruído.

## 💸 Destino dos BNB Arrecadados

Todo BNB recebido segue uma regra fixa:

- **90%** → Liquidity Bootstrapper  
- **10%** → Fundador (custos operacionais, infraestrutura, continuidade)

Essa divisão:
- É conhecida antes da pré-venda
- Está codificada no contrato
- Não pode ser alterada

## 🤖 Proteções Anti-Abuso

A pré-venda inclui proteções como:

- Limite por carteira (max buy)
- Preço fixo e previsível
- Execução direta on-chain
- Sem acesso privilegiado
- Sem função de saque manual

Bots não têm vantagem estrutural.  
Humanos não têm privilégios ocultos.

## 🧭 Integração com o Ecossistema

O contrato de pré-venda:

- Resolve dependências via **MIMHO Registry**
- Envia tokens para **Vesting**
- Envia BNB para **Liquidity Bootstrapper**
- Emite eventos no **Events Hub**
- Não cria liquidez sozinho
- Não controla launch

Ele faz **uma coisa** e faz bem.

## 🏛️ Governança

- Antes da DAO: controle do fundador
- Após a DAO: contrato permanece imutável
- A DAO **não** pode mudar regras da pré-venda
- Nenhuma votação altera parâmetros já executados

A pré-venda não é governável.  
Ela é **executável e final**.

Para o projeto:
- Liquidez garantida

- Início limpo

Para auditoria:
