# Security Policy ‚Äî MIMHO Ecosystem

The MIMHO Ecosystem takes security seriously.  
If you believe you have found a security vulnerability, please report it responsibly.

---

## ‚úÖ Supported Scope

This repository may contain:
- Technical specifications (`/contracts/specs`)
- Address registries (`/addresses`)
- Public documentation (`/docs`)
- Solidity placeholders (`/contracts/solidity`) and/or verified Solidity sources after official deployment

Reports are welcome for:
- Smart contract vulnerabilities (logic, access control, reentrancy, overflow/underflow, approval issues, etc.)
- Registry/address integrity problems (incorrect addresses, swapped wallets, malicious modifications)
- Website or documentation vulnerabilities that may impact users (phishing vectors, injected scripts, etc.)
- Supply-chain or dependency vulnerabilities affecting the project

## ü§ù Responsible Disclosure

Please do not:

- Engage in social engineering, phishing, or doxxing

If the report is valid and responsibly disclosed, we will credit the reporter (optional) in our release notes or documentation.
